# Database Schema

## Security & Row-Level Security (RLS)

All tables strictly enforce PostgreSQL Row-Level Security.
- Users are mapped via Clerk JWT claims to the `auth.uid()`.
- Projects are isolated: `user_id = auth.uid()`.
- Papers and Activities cascade their RLS through `project_id`.

## Vector Search Index (Qdrant)

While relational data lives in Supabase, vector embeddings live in Qdrant Cloud.
- **Payload Schema**: `{ paperId: UUID, projectId: UUID, content: string, pageNumber: number }`
- **Isolation**: Qdrant queries ALWAYS apply an explicit payload filter `must: [{ key: "projectId", match: { value: <UUID> } }]` to prevent cross-tenant vector contamination.

## Supabase Tables

### 1. `projects`
- `id` (UUID, Primary Key)
- `user_id` (String) - Maps to Clerk authentication `user_id`
- `title` (String)
- `description` (String, Nullable)
- `status` (String) - 'active', 'archived', 'deleted'
- `is_favorite` (Boolean)
- `created_at` (Timestamp)
- `updated_at` (Timestamp)

### 2. `papers`
- `id` (UUID, Primary Key)
- `project_id` (UUID, Foreign Key -> projects.id)
- `user_id` (String)
- `title` (String)
- `file_url` (String) - Public or signed URL to the stored PDF
- `file_name` (String) - Original name of the uploaded PDF
- `status` (String) - 'queued', 'processing', 'embedding', 'summarizing', 'completed', 'failed'
- `summary` (JSONB, Nullable) - Extracted summary including abstract, methodology, datasets, and findings
- `authors` (JSONB)
- `created_at` (Timestamp)
- `updated_at` (Timestamp)

### 3. `activities`
- `id` (UUID, Primary Key)
- `project_id` (UUID, Foreign Key -> projects.id)
- `type` (String) - e.g., 'project_created', 'paper_added', 'summary_generated', 'chat_started'
- `description` (String)
- `created_at` (Timestamp)

### 4. `conversations` & `messages`
- Track the RAG chat history for the AI Copilot.
- **messages**: `id`, `conversation_id`, `project_id`, `role` ('user', 'assistant', 'system'), `content`, `created_at`.

### 5. `comparisons`
- `id` (UUID, Primary Key)
- `project_id` (UUID, Foreign Key -> projects.id)
- `paper_ids` (Array of UUIDs)
- `theme` (String)
- `common_findings` (JSONB)
- `contrasting_results` (JSONB)
- `research_gaps` (JSONB)
- `created_at` (Timestamp)

*(Note: Users are implicitly managed via Clerk IDs, eliminating the need for a dedicated local users table.)*

## Qdrant (Vector Data)

### Collection: `litlens_paper_chunks`
- **id**: Unique chunk UUID
- **vector**: `1536`-dimensional vector generated by OpenAI (`text-embedding-3-small`)
- **payload** (Metadata):
  - `paper_id` (Keyword) - Maps to PostgreSQL `papers.id` for filtering by specific papers.
  - `project_id` (Keyword) - Maps to PostgreSQL `projects.id` for cross-paper search within a project.
  - `chunk_index` (Integer) - Ordering of the chunk within the document.
  - `text` (Text) - The raw text content of the chunk (not indexed, stored purely for RAG retrieval).
