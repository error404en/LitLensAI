# Database Schema

## Supabase PostgreSQL (Relational Data)

### 1. `projects`
- `id` (UUID, Primary Key)
- `user_id` (String) - Maps to Clerk authentication `user_id`
- `title` (String)
- `description` (String, Nullable)
- `created_at` (Timestamp)
- `updated_at` (Timestamp)

### 2. `documents`
- `id` (UUID, Primary Key)
- `project_id` (UUID, Foreign Key -> projects.id)
- `user_id` (String)
- `file_url` (String) - Public or signed URL to the stored PDF
- `file_name` (String) - Original name of the uploaded PDF
- `file_size` (Integer, Nullable)
- `mime_type` (String, Nullable)
- `status` (String) - 'queued', 'processing', 'embedding', 'summarizing', 'completed', 'failed'
- `metadata` (JSONB, Nullable) - Extracted metadata from LangChain (e.g., abstract, methodology, datasets, findings)
- `created_at` (Timestamp)
- `updated_at` (Timestamp)

*(Note: Users are implicitly managed via Clerk IDs, eliminating the need for a dedicated local users table unless specific profile extensions are required.)*

## Qdrant (Vector Data)

### Collection: `litlens_paper_chunks`
- **id**: Unique chunk UUID
- **vector**: `1536`-dimensional vector generated by OpenAI (`text-embedding-3-small`)
- **payload** (Metadata):
  - `paper_id` (Keyword) - Maps to PostgreSQL `documents.id` for filtering by specific papers.
  - `project_id` (Keyword) - Maps to PostgreSQL `projects.id` for cross-paper search within a project.
  - `chunk_index` (Integer) - Ordering of the chunk within the document.
  - `text` (Text) - The raw text content of the chunk (not indexed, stored purely for RAG retrieval).
