# Database Schema

## PostgreSQL (Relational Data)

### 1. `users`
- `id` (UUID, Primary Key)
- `clerk_id` (String, Unique) - Maps to Clerk authentication
- `email` (String)
- `created_at` (Timestamp)

### 2. `projects`
- `id` (UUID, Primary Key)
- `user_id` (UUID, Foreign Key -> users.id)
- `title` (String)
- `description` (Text)
- `created_at` (Timestamp)

### 3. `documents`
- `id` (UUID, Primary Key)
- `project_id` (UUID, Foreign Key -> projects.id)
- `title` (String)
- `file_path` (String) - Path to the stored PDF
- `status` (String) - e.g., 'processing', 'completed', 'failed'
- `summary` (Text)
- `methodology` (Text)
- `datasets` (Text)
- `findings` (Text)
- `created_at` (Timestamp)

## ChromaDB (Vector Data)

### Collection: `document_chunks`
- **id**: Unique chunk ID (e.g., `doc_id_chunk_index`)
- **embedding**: Vector generated by OpenAI (`text-embedding-3-small`)
- **document**: The raw text content of the chunk
- **metadata**:
  - `document_id` (UUID, maps to PostgreSQL documents.id)
  - `project_id` (UUID)
  - `page_number` (Integer)
